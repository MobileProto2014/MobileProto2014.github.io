Share your app ideas with us and explain the target user you have in mind and the API you plan on using.

For those of you making HTTP requests, use the Android Volley Library. https://github.com/mcxiaoke/android-volley
You can include the dependency via Gradle.
This library will help you make HTTP requests and manage them for you. You can pass in callbacks to handle the return data and handle different responses. It will handle running these requests off the mainUI thread so your app doesn't freeze when making requests. It's also good at managing cache, image downloading, threading, etc. Read more about it online (it's pretty popular).

Upcoming Deliverables:
For Monday, 9/29 you should have a chat room that pushes and pulls chats from Firebase with a specific room ID (name it whatever you want but don’t change it after you’ve chosen one).
You want to push Java objects with the attributes: name, message, timestamp. Everything is in strings.
to the following URL: https://mobileproto2014.firebaseio.com/chatroom/[[CHATROOM_ID]]
myFireBaseRef.push( [[OBJECT_HERE]] )
This will create a unique ID for your object and appends it to all the previously pushed chatroom data, you do not overwrite anything (you can very, very easily). You can find more details under “Guide” part of Android tutorial – or check out a few of the others on the Firebase website.
When you pull from a chatroom (it will update automatically), you will be getting an Array of Java Objects.
